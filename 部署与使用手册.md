## 一、本地运行

### 1.1 环境要求

| 工具 | 版本要求 | 下载地址 |
|------|---------|---------|
| Node.js | >= 22 | https://nodejs.org |
| pnpm | >= 9 | `corepack enable && corepack prepare pnpm@latest --activate` |
| PostgreSQL | >= 16 | https://www.postgresql.org/download/ |
| Git | 任意 | https://git-scm.com |

### 1.2 克隆项目

```bash
git clone <仓库地址> graduate_school
cd graduate_school
```

### 1.3 配置数据库

#### Windows 安装 PostgreSQL（如未安装）

1. 从 https://www.postgresql.org/download/windows/ 下载安装包
2. 安装时记下设置的 postgres 用户密码
3. 安装完成后打开 **SQL Shell (psql)**，执行：

```sql
CREATE DATABASE class_memories;
```

或用命令行：

```bash
createdb -U postgres class_memories
```

### 1.4 配置环境变量

```bash
# 后端配置
cp backend/.env.example backend/.env
```

编辑 `backend/.env`：

```env
DATABASE_URL="postgresql://postgres:你的密码@localhost:5432/class_memories?schema=public"
JWT_SECRET="生成一个随机字符串，越长越好"
JWT_EXPIRES_IN="7d"
PORT=3000
DEEPSEEK_API_KEY="你的DeepSeek API Key（可选，AI功能需要）"
```

### 1.5 启动后端

```bash
cd backend

# 安装依赖
pnpm install

# 生成 Prisma 客户端 + 同步数据库表
npx prisma generate
npx prisma db push

# 创建管理员账号（可选）
$env:ADMIN_PASSWORD="admin123"   # Windows PowerShell
# 或 export ADMIN_PASSWORD="admin123"  # Linux/Mac
npx prisma db seed

# 启动开发服务器
pnpm start:dev
```

后端启动在 **http://localhost:3000**，API 文档在 **http://localhost:3000/docs**。

### 1.6 启动前端

```bash
# 新开一个终端
cd frontend

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

前端访问 **http://localhost:5173**。

### 1.7 验证运行状态

```bash
# 测试后端健康检查
curl http://localhost:3000/api/health

# 应返回类似：
# {"status":"ok","timestamp":"2026-06-02T..."}
```

### 1.8 常用命令

```bash
# 后端
pnpm start:dev     # 开发模式（热重载）
pnpm build         # 构建
pnpm start:prod    # 生产模式启动
pnpm test          # 运行测试

# 前端
pnpm dev           # 开发模式
pnpm build         # 构建
pnpm test          # 运行测试
```

### 1.9 默认管理员账号

执行 seed 脚本后（需设置 `ADMIN_PASSWORD` 环境变量）：

| 用户名 | 密码 |
|--------|------|
| `admin` | `ADMIN_PASSWORD` 环境变量的值 |

## 二、服务器部署

### 2.1 购买服务器

**推荐云服务商：**

| 服务商 | 优势 | 新用户优惠 |
|--------|------|-----------|
| 阿里云 | 国内访问快，文档齐全 | 轻量应用服务器 ~50元/月起 |
| 腾讯云 | 与微信生态集成好 | 轻量应用服务器 ~50元/月起 |
| 华为云 | 稳定，教育优惠多 | 学生优惠 ~30元/月起 |
| UCloud | 按量计费灵活 | 国内用户免备案线路 |

**推荐配置（轻量应用服务器）：**

| 配置 | 价格 | 适用场景 |
|------|------|---------|
| 2 核 2G | ~50 元/月 | 10-30 人小班使用 |
| 2 核 4G | ~80 元/月 | 50 人以上，带 AI 功能 |
| 4 核 8G | ~150 元/月 | 高并发，大量照片视频 |

**选择 Ubuntu 24.04 LTS 系统**，开放端口：`22`(SSH)、`80`(HTTP)、`443`(HTTPS)、`3000`(API)。

> **提示**：如果暂时不想购买服务器，可以先跳到[第三节](#三内网穿透cloudflare-tunnel)用内网穿透让同学从外网访问你的本地电脑。

### 2.2 域名购买与解析

推荐域名注册商：

| 服务商 | 价格 | 特点 |
|--------|------|------|
| 阿里云万网 | 首年 ~30 元 | 与阿里云服务器配合好 |
| 腾讯云 DNSPod | 首年 ~30 元 | 免费 DNS 解析 |
| Cloudflare | 按成本价 | 自带 CDN + DDoS 防护 |
| NameSilo | ~9 美元/年 | 永久免费 WHOIS 隐私保护 |

**域名建议：**
- 使用有纪念意义的域名，如 `class2023.com`、`ourclass.cn`
- 或使用 `班级名.中文` 的中文域名
- 如果只通过 IP 访问，可以跳过域名步骤，但不能配置 HTTPS

**DNS 解析（在域名注册商的控制台操作）：**

```
记录类型: A
主机记录: @       (表示根域名，如 class2023.com)
记录值:   你的服务器公网 IP
TTL:     10 分钟

记录类型: A
主机记录: www     (表示 www.class2023.com)
记录值:   你的服务器公网 IP
TTL:     10 分钟
```

配置后等待 1-10 分钟生效，用 `ping 你的域名.com` 验证。

### 2.3 服务器初始化

通过 SSH 连接到服务器：

```bash
ssh root@你的服务器IP
```

首次连接会提示确认指纹，输入 `yes` 即可。

**执行初始化脚本：**

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | bash

# 安装 Docker Compose 插件
apt install -y docker-compose-plugin

# 安装 Nginx
apt install -y nginx

# 验证安装
docker --version          # 应输出 Docker 版本号
docker compose version     # 应输出 Docker Compose 版本号
nginx -v                   # 应输出 Nginx 版本号
```

> **注意**：如果在中国大陆的服务器上安装 Docker，可能会遇到 `get.docker.com` 连接慢的问题。可以使用镜像加速：
> ```bash
> curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
> echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" > /etc/apt/sources.list.d/docker.list
> apt update && apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
> ```

### 2.5 配置 Docker Compose

```bash
ssh root@你的服务器IP
cd /opt/class_memories/backend
```

编辑 `docker-compose.yml`（确认已存在，可按需修改）：

```bash
nano docker-compose.yml
```

确认内容如下（如没有则创建）：

```yaml
version: '3.8'
services:
  postgres:
    image: postgres:16-alpine
    container_name: class_memories_db
    restart: unless-stopped
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: 你的数据库密码
      POSTGRES_DB: graduate_school
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U app -d graduate_school']
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    build: .
    container_name: class_memories_app
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      DATABASE_URL: postgresql://app:你的数据库密码@postgres:5432/graduate_school?schema=public
      JWT_SECRET: 你的JWT密钥
      JWT_EXPIRES_IN: 7d
      PORT: 3000
      DEEPSEEK_API_KEY: 你的DeepSeek API Key（可选）
    ports:
      - '127.0.0.1:3000:3000'  # 仅监听本地，通过 Nginx 代理
    volumes:
      - uploads:/app/uploads

volumes:
  postgres_data:
  uploads:
```

> **关键说明**：
> - `ports: '127.0.0.1:3000:3000'` 表示后端只监听本地，不直接暴露到公网，通过 Nginx 反向代理访问
> - 务必修改 `POSTGRES_PASSWORD` 和 `JWT_SECRET` 为随机字符串
> - `JWT_SECRET` 可以使用命令生成：`openssl rand -hex 32`

### 2.6 配置生产环境变量

也可以使用 `.env` 文件方式（如果 docker-compose.yml 引用 ${} 变量）：

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env
nano .env
```

生产环境 `.env` 配置：

```env
# 数据库连接（使用 Docker Compose 内的 PostgreSQL）
DATABASE_URL="postgresql://app:你的密码@postgres:5432/graduate_school?schema=public"

# JWT 密钥（务必修改为随机字符串，至少 32 位）
JWT_SECRET="这里填一个至少32位的随机字符串"

# Token 有效期
JWT_EXPIRES_IN="7d"

# 后端端口
PORT=3000

# 可选：DeepSeek API Key（用于 AI 树洞功能）
DEEPSEEK_API_KEY="你的DeepSeek API Key"
```

> **注意**：生产环境 `.env` 中的 `DATABASE_URL` 的 host 部分应为 `postgres`（Docker Compose 的服务名），而不是 `localhost`。因为 app 容器和 postgres 容器在同一个 Docker 网络中通信。

### 2.7 配置 Nginx 反向代理

创建 Nginx 站点配置：

```bash
nano /etc/nginx/sites-available/class-memories
```

粘贴以下内容（替换 `你的域名.com` 为实际域名）：

```nginx
server {
    listen 80;
    server_name 你的域名.com www.你的域名.com;

    # 前端静态文件
    root /opt/class_memories/frontend/dist;
    index index.html;

    # 日志
    access_log /var/log/nginx/class-memories-access.log;
    error_log /var/log/nginx/class-memories-error.log;

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置（上传大文件时需要）
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;

        # 最大请求体（照片上传）
        client_max_body_size 50M;
    }

    # WebSocket 代理（用于实时通知）
    location /ws/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 需要更长的超时
        proxy_read_timeout 3600s;
    }

    # 上传文件代理（图片等静态资源缓存）
    location /uploads/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_cache_valid 200 1d;
        add_header Cache-Control "public, immutable";
        expires 30d;
    }

    # SPA 路由：所有非文件请求返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

启用站点：

```bash
# 创建软链接启用站点
ln -s /etc/nginx/sites-available/class-memories /etc/nginx/sites-enabled/

# 删除默认站点
rm /etc/nginx/sites-enabled/default

# 测试配置是否正确
nginx -t

# 重启 Nginx
systemctl restart nginx

# 查看 Nginx 状态
systemctl status nginx
```

如果 `nginx -t` 报错，根据错误提示检查配置文件的语法。

### 2.10 初始化数据库

在服务启动后，执行数据库迁移和种子数据：

```bash
cd /opt/class_memories/backend

# 执行数据库迁移（创建表结构）
docker compose exec app npx prisma db push

# 创建管理员账号（先设置密码环境变量）
docker compose exec -e ADMIN_PASSWORD="admin123" app npx prisma db seed

```

> **默认管理员账号**：用户名 `admin`，密码为上面 `ADMIN_PASSWORD` 设置的值（示例中是 `admin123`）。部署后请立即修改密码。

### 2.11 配置 HTTPS（使用 Let's Encrypt）

```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 申请证书并自动配置 Nginx
certbot --nginx -d 你的域名.com -d www.你的域名.com

# 按照提示：
# 1. 输入邮箱地址（用于接收续期提醒）
# 2. 同意服务条款
# 3. 选择是否将 HTTP 重定向到 HTTPS（选 2，推荐）

# 验证证书自动续期
certbot renew --dry-run
```

> **HTTPS 配置完成后**，Nginx 配置会被 Certbot 自动修改，`listen 80` 旁边会添加 `listen 443 ssl` 和证书路径。之后访问 `https://你的域名.com` 即可。

### 2.12 配置 CORS（跨域）

编辑后端 `backend/src/main.ts`，修改 CORS 配置：

```typescript
// 找到 cors 配置部分
app.enableCors({
  origin: [
    'http://localhost:5173',           // 本地开发
    'http://localhost:3000',
    'https://你的域名.com',             // 你的域名
    'https://www.你的域名.com',
  ],
  credentials: true,
});
```

重新构建并重启：

```bash
cd /opt/class_memories/backend
docker compose build app
docker compose up -d
```

### 2.16 故障排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| `docker compose up -d` 后服务立即退出 | 数据库连接失败 | 检查 `.env` 中 `DATABASE_URL` 的密码是否正确 |
| Nginx 返回 502 Bad Gateway | 后端未启动或端口配置错误 | `docker compose ps` 确认 app 在运行；检查 Nginx `proxy_pass` 端口 |
| Nginx 返回 413 Request Entity Too Large | 上传文件超过 Nginx 限制 | 在 Nginx 配置中添加 `client_max_body_size 50M;` |
| 前端白屏，控制台 404 | Nginx 未配置 SPA 路由 | 确认 `try_files $uri $uri/ /index.html;` 已配置 |
| HTTPS 证书申请失败 | 域名未正确解析到服务器 | `ping 你的域名.com` 确认 IP 正确 |
| 上传图片后无法显示 | uploads 目录权限问题 | `docker compose exec app chmod 777 uploads` |
| Docker 构建太慢 | 大陆服务器访问 Docker Hub 慢 | 配置 Docker 镜像加速器 |

## 三、内网穿透（Cloudflare Tunnel）

如果你没有云服务器，或者想先让班级同学从外网访问你的本地电脑，可以用 Cloudflare Tunnel 实现内网穿透。

### 3.1 工作原理

```
同学浏览器 → https://api.你的域名.com → Cloudflare 边缘节点 → Cloudflare Tunnel → 你的本地电脑:3000
```

- 你的电脑主动**向外出**连接 Cloudflare 的边缘节点，所以即使没有公网 IP 也能工作
- Cloudflare 负责全球加速和 DDoS 防护
- 所有流量都经过 HTTPS 加密，安全可靠

### 3.2 准备工作

1. 注册 Cloudflare 账号：https://dash.cloudflare.com/signup
2. 拥有一个域名，并将 DNS 托管到 Cloudflare（在 Cloudflare 控制台添加域名，按提示修改域名的 NS 记录）
3. 确保本地电脑可以长时间开机运行

> **DNS 托管步骤**（如果域名不在 Cloudflare）：
> 1. 在 Cloudflare 控制台点击 "Add a Site"，输入你的域名
> 2. Cloudflare 会扫描现有的 DNS 记录并自动导入
> 3. Cloudflare 会分配两个 NS 服务器地址（如 `daisy.ns.cloudflare.com`）
> 4. 到你购买域名的注册商控制台，将域名的 NS 记录修改为 Cloudflare 的地址
> 5. 等待 1-24 小时生效

### 3.3 安装 Cloudflare Tunnel

- **请记下这个 Tunnel ID**，后面需要用到

### 3.7 配置隧道

创建或编辑配置文件：

```powershell
notepad C:\Users\你的用户名\.cloudflared\config.yml
```

写入以下内容（替换 `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` 为你的 Tunnel ID）：

```yaml
tunnel: a1b2c3d4-e5f6-7890-abcd-ef1234567890
credentials-file: C:\Users\你的用户名\.cloudflared\a1b2c3d4-e5f6-7890-abcd-ef1234567890.json

ingress:
  # 前端 + API 全部走同一个域名
  - hostname: api.你的域名.com
    service: http://localhost:3000
    originRequest:
      noTLSVerify: true

  # WebSocket（实时通知）
  - hostname: ws.你的域名.com
    service: http://localhost:3000

  # 兜底规则：不匹配以上规则的返回 404
  - service: http_status:404
```

> **配置说明**：
> - `ingress` 定义了不同域名指向的本地服务
> - 这里把 `api.你的域名.com` 指向本地 3000 端口（NestJS 后端）
> - 前端也会由后端 ServeStaticModule 提供静态文件，所以不需要单独配置前端
> - WebSocket 单独分配一个子域名用于实时通知

### 5.12 开启/维护模式

临时关闭网站进行维护：

**方式一：Nginx 返回维护页面（服务器部署）**

```nginx
# /etc/nginx/sites-available/class-memories
server {
    listen 80;
    server_name 你的域名.com;

    # 维护模式：返回一个简单的维护页面
    location / {
        return 503;
    }

    # 自定义 503 页面
    error_page 503 /maintenance.html;
    location = /maintenance.html {
        root /opt/class_memories/frontend/dist;
    }
}
```

创建 `maintenance.html`：

```html
<!DOCTYPE html>
<html>
<head><title>维护中</title></head>
<body style="display:flex;justify-content:center;align-items:center;height:100vh;font-family:sans-serif;">
  <div style="text-align:center;">
    <h1>🚧 站点维护中</h1>
    <p>稍后回来，我们正在升级功能 :)</p>
  </div>
</body>
</html>
```

**方式二：后端维护开关**

在 `backend/.env` 中添加：

```env
MAINTENANCE_MODE=false
```

然后在后端添加一个全局中间件检查：

```typescript
// backend/src/main.ts
if (process.env.MAINTENANCE_MODE === 'true') {
  app.use('/api', (req, res) => {
    res.status(503).json({ message: '站点维护中' });
  });
}
```

### 5.13 修改速率限制

限流配置在 `backend/src/main.ts`：

```typescript
// 全局限流
app.use(
  rateLimit({
    windowMs: 60 * 1000,  // 1分钟窗口
    max: 60,               // 每分钟最多 60 个请求
    message: '请求过于频繁，请稍后再试',
  }),
);

// 登录限流（更严格）
// 在 auth.controller.ts 中
@Throttle({ default: { limit: 20, ttl: 60000 } })  // 每分钟最多 20 次登录尝试
```

**可按需调整：**

| 场景 | 建议值 |
|------|--------|
| 正常浏览 | 60 次/分钟 |
| 登录接口 | 10-20 次/分钟 |
| 注册接口 | 5-10 次/分钟 |
| AI 对话 | 20 次/分钟 |
