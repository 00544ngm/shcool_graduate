# Class Memories V2.0 完整开发蓝图

## 项目总目标
打造一个面向毕业班级、社团和校友组织的数字时光馆平台，支持照片、视频、时间轴、留言、未来信箱、AI回忆助手等能力。


## 第1章：产品愿景与定位

### 产品愿景与定位 - 子章节1
目标：定义产品愿景与定位的核心职责、边界与扩展能力。

- 功能说明
- 用户价值
- 技术方案
- 数据模型
- API设计
- 风险分析
- 扩展路线

详细描述：
产品愿景与定位模块采用高内聚低耦合设计，所有业务能力通过独立模块封装。模块之间通过接口和事件通信，避免跨模块直接依赖。前端采用特性分层结构，后端采用领域驱动设计。每个聚合根拥有独立仓储、服务和控制器，实现长期可维护性。

# Claude Code 执行规范

## 强制要求

1. TypeScript Strict Mode
2. ESLint + Prettier
3. 单模块单职责
4. 禁止循环依赖
5. API统一响应格式
6. Docker化部署
7. 单元测试覆盖核心业务
8. 所有配置通过环境变量管理

Phase 1：认证系统
Phase 2：照片墙
Phase 3：留言系统
Phase 4：时间轴
Phase 5：视频馆
Phase 6：未来信箱
Phase 7：AI助手
Phase 8：性能优化
Phase 9：部署上线

## 最终目标

构建一个毕业十年后仍然能够访问、浏览、留言和沉淀记忆的数字时光馆。
